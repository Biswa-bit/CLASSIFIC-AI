# Distribution Analysis

---

# Overview

Distribution Analysis is one of the core modules within the CLASSIFIC-AI Data Profiling Agent.

The purpose of this module is to analyze how numerical data is distributed across each feature. Understanding the distribution of data is essential for selecting appropriate preprocessing techniques, statistical methods, machine learning algorithms, and evaluation strategies.

This module evaluates the shape, spread, symmetry, and concentration of numerical features using descriptive statistics and statistical tests.

The results generated by Distribution Analysis are later consumed by Outlier Detection, Scaling Recommendation, Feature Engineering, Business Rules Agent, and the Profiling Recommendation Engine.

---

# Purpose

The objectives of the Distribution Analysis module are:

- Understand feature distributions
- Detect skewed variables
- Measure kurtosis
- Identify normal distributions
- Recommend suitable preprocessing
- Improve model performance
- Support explainable AI decisions
- Guide feature engineering

---

# Why Distribution Analysis is Important

Machine learning algorithms often make assumptions about data distributions.

Understanding feature distributions helps determine:

- Whether scaling is required
- Whether transformations are needed
- Whether outlier detection should be performed
- Whether statistical assumptions are satisfied
- Whether robust algorithms should be preferred

Proper distribution analysis improves model accuracy and interpretability.

---

# Distribution Characteristics Evaluated

The module evaluates several statistical properties for every numerical feature.

### Distribution Shape

Determines whether the data is:

- Normal
- Left-Skewed
- Right-Skewed
- Uniform
- Multimodal
- Exponential

---

### Central Tendency

Evaluates:

- Mean
- Median
- Mode

Differences between these values often indicate skewness.

---

### Dispersion

Measures:

- Standard Deviation
- Variance
- Range
- Interquartile Range (IQR)

---

### Skewness

Measures the symmetry of the distribution.

Interpretation

| Skewness | Interpretation |
|----------|----------------|
| -0.5 to 0.5 | Approximately Symmetric |
| < -0.5 | Left Skewed |
| > 0.5 | Right Skewed |

---

### Kurtosis

Measures the heaviness of distribution tails.

Interpretation

| Kurtosis | Interpretation |
|----------|----------------|
| ≈ 3 | Normal Distribution |
| > 3 | Heavy Tails |
| < 3 | Light Tails |

---

### Normality

The module evaluates whether data approximately follows a normal distribution.

Future versions will include statistical tests such as:

- Shapiro-Wilk Test
- Anderson-Darling Test
- Kolmogorov-Smirnov Test

# CLASSIFIC-AI Decision Logic

The Distribution Analysis module applies predefined decision rules.

| Condition | Recommendation |
|-----------|----------------|
| Approximately Normal | StandardScaler |
| Right Skewed | Log Transformation |
| Left Skewed | Power Transformation |
| Heavy Tails | RobustScaler |
| Extreme Skewness | Outlier Detection |
| Non-Normal Distribution | Consider Robust Models |

# Common Mistakes

Avoid:

- Assuming all variables are normally distributed

- Ignoring skewness

- Ignoring kurtosis

- Applying StandardScaler to highly skewed data

- Transforming variables without business justification
