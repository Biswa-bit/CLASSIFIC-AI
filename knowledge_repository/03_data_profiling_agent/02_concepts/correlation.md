# Correlation

---

# Overview

Correlation measures the strength and direction of the relationship between two variables.

Understanding feature relationships is one of the most important tasks during data profiling because highly correlated variables may introduce redundancy, increase multicollinearity, and reduce model interpretability.

Within CLASSIFIC-AI, the Correlation Analysis module automatically evaluates relationships between numerical features and identifies variables that may require further investigation before model development.

The outputs generated by this module support intelligent preprocessing recommendations, feature selection, explainable AI, and model optimization.

---

# Definition

Correlation is a statistical measure that quantifies how strongly two variables move together.

Correlation values range from:

- -1 → Perfect Negative Correlation
- 0 → No Correlation
- +1 → Perfect Positive Correlation

The closer the correlation coefficient is to ±1, the stronger the relationship between the variables.

---

# Objectives

The objectives of Correlation Analysis are:

- Identify relationships between variables
- Detect redundant features
- Detect multicollinearity
- Improve feature selection
- Improve model performance
- Support explainable AI

---

# Why Correlation is Important

Highly correlated features often contain similar information.

Using redundant variables may:

- Increase multicollinearity
- Reduce model stability
- Increase computational complexity
- Reduce explainability

Correlation analysis helps identify which variables provide unique information and which may be removed or combined.

---

# Types of Correlation

## Positive Correlation

As one variable increases, the other also increases.

Example:

- Experience ↑
- Salary ↑

---

## Negative Correlation

As one variable increases, the other decreases.

Example:

- Price ↑
- Demand ↓

---

## No Correlation

No meaningful relationship exists between the variables.

Example:

- Shoe size
- Stock market returns

---

# Correlation Coefficient Interpretation

| Correlation | Interpretation |
|-------------|----------------|
| 0.00 – 0.19 | Very Weak |
| 0.20 – 0.39 | Weak |
| 0.40 – 0.59 | Moderate |
| 0.60 – 0.79 | Strong |
| 0.80 – 1.00 | Very Strong |

The same interpretation applies for negative values.

---

# Correlation Methods

CLASSIFIC-AI supports multiple correlation techniques.

## Pearson Correlation

Measures linear relationships.

Recommended for:

- Continuous variables
- Normally distributed data

---

## Spearman Rank Correlation

Measures monotonic relationships.

Recommended for:

- Non-normal distributions
- Ordinal variables
- Outlier-resistant analysis

---

## Kendall Tau

Measures rank agreement.

Recommended for:

- Small datasets
- Ordinal data
- Robust rank comparison

# Correlation Matrix

A correlation matrix displays pairwise correlations between all numerical variables.

It helps identify:

- Highly correlated features
- Redundant variables
- Candidate variables for feature selection

CLASSIFIC-AI automatically generates a correlation matrix during data profiling.

---

# Heatmap

A heatmap provides a visual representation of the correlation matrix.

Benefits include:

- Quick identification of highly correlated variables
- Easier interpretation
- Better feature selection decisions

---

# Multicollinearity

Multicollinearity occurs when multiple predictor variables are highly correlated.

Problems caused by multicollinearity include:

- Unstable regression coefficients
- Increased variance
- Poor interpretability
- Reduced statistical significance

---

# Variance Inflation Factor (VIF)

CLASSIFIC-AI uses VIF to measure multicollinearity.

General interpretation:

| VIF | Interpretation |
|-----|----------------|
| 1 | No correlation |
| 1–5 | Acceptable |
| 5–10 | Moderate multicollinearity |
| >10 | High multicollinearity |

Variables with very high VIF values may be recommended for removal.

---

# CLASSIFIC-AI Decision Logic

The Correlation module recommends additional preprocessing when:

- Absolute correlation exceeds the defined threshold
- VIF exceeds acceptable limits
- Redundant variables are detected
- Multiple variables represent similar business information

Possible recommendations include:

- Remove redundant features
- Perform feature selection
- Apply Principal Component Analysis (Future)
- Retain variables based on business importance

# CLASSIFIC-AI Recommendation Logic

| Observation | Recommendation |
|-------------|----------------|
| Low Correlation | Keep both variables |
| Moderate Correlation | Monitor |
| High Correlation | Review business importance |
| Very High Correlation | Consider removing one feature |
| High VIF | Remove or combine variables |
| Redundant Information | Feature Selection |

---

# Human Approval

Correlation-based recommendations are never applied automatically.

The user decides whether to:

- Remove variables
- Keep correlated variables
- Apply business-specific rules
- Perform additional analysis

---

# Best Practices

✔ Always inspect the correlation matrix

✔ Use heatmaps for visualization

✔ Consider business meaning before removing variables

✔ Evaluate VIF alongside correlation

✔ Use feature importance before dropping variables

---

# Common Mistakes

Avoid:

- Removing variables using correlation alone

- Ignoring business importance

- Using Pearson correlation for ordinal variables

- Ignoring nonlinear relationships

- Assuming correlation implies causation
