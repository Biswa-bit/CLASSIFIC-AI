# Normality

---

# Overview

Normality refers to whether a dataset follows a Normal (Gaussian) Distribution.

Many classical statistical techniques assume that numerical variables are normally distributed. Therefore, assessing normality is an important step during data profiling before performing feature engineering, statistical inference, or machine learning.

Within CLASSIFIC-AI, the Normality module automatically evaluates numerical features to determine whether they approximately follow a normal distribution and recommends appropriate preprocessing techniques when deviations are detected.

The outputs generated by this module are used throughout the CLASSIFIC-AI pipeline to support intelligent preprocessing recommendations, model selection, and explainable AI.

---

# Definition

A variable is considered normally distributed when its values form a symmetric, bell-shaped curve around the mean.

Characteristics include:

- Mean ≈ Median ≈ Mode
- Symmetric distribution
- Skewness close to zero
- Most observations concentrated around the center
- Gradually decreasing frequency toward both tails

Normality allows many statistical methods to perform efficiently and reliably.

# Why Normality is Important

Many statistical models assume normally distributed variables.

Evaluating normality helps determine:

- Whether transformation is required
- Whether parametric tests are appropriate
- Whether robust statistical methods should be used
- Whether scaling techniques are appropriate
- Whether assumptions of certain machine learning algorithms are satisfied

Normality analysis improves model reliability and interpretability.

---

# Characteristics of a Normal Distribution

A normal distribution has the following characteristics:

- Bell-shaped curve
- Symmetric around the mean
- Mean = Median = Mode
- Skewness ≈ 0
- Kurtosis ≈ 3 (Mesokurtic)
- Approximately 68% of observations lie within one standard deviation
- Approximately 95% lie within two standard deviations
- Approximately 99.7% lie within three standard deviations

This is known as the **68–95–99.7 Rule**.

## Statistical Tests

- Shapiro-Wilk Test
- Kolmogorov-Smirnov Test
- Anderson-Darling Test
- D'Agostino-Pearson Test

Future versions will automatically compare multiple normality tests.

---

# Interpretation

If the p-value is greater than the chosen significance level (commonly 0.05):

- Data is considered approximately normal.

If the p-value is less than the significance level:

- Data significantly deviates from a normal distribution.

The result should always be interpreted together with visualizations.

---

# CLASSIFIC-AI Decision Logic

The Normality module recommends additional preprocessing when:

- Normality tests fail
- Strong skewness is detected
- Heavy tails are present
- Extreme outliers influence the distribution

Possible recommendations include:

- Log Transformation
- Square Root Transformation
- Box-Cox Transformation
- Yeo-Johnson Transformation
- Robust Scaling

# CLASSIFIC-AI Recommendation Logic

| Observation | Recommendation |
|-------------|----------------|
| Normally Distributed | No transformation required |
| Mild Skewness | Monitor only |
| Moderate Skewness | Consider Log Transformation |
| Severe Skewness | Box-Cox or Yeo-Johnson |
| Heavy Tails | RobustScaler |
| Significant Outliers | Perform Outlier Detection |

# Human Approval

Normality-based recommendations are never applied automatically.

The user decides whether to:

- Accept transformation
- Ignore recommendation
- Apply business-specific preprocessing
- Perform additional statistical testing

# Best Practices

✔ Use both visual inspection and statistical tests

✔ Consider sample size before interpreting p-values

✔ Investigate the causes of non-normality

✔ Apply transformations only when appropriate

✔ Consider business meaning before modifying variables

---

# Common Mistakes

Avoid:

- Relying solely on statistical tests

- Assuming every feature must be normally distributed

- Ignoring visual inspection

- Applying unnecessary transformations

- Ignoring sample size effects
